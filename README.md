## ⚡ Быстрый старт

Скопируйте и вставьте одну команду на сервер:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/sxcvio/Auto-MTProto/main/mtproxy-install.sh)
```

## 🔧 Что делает скрипт

| Шаг | Действие |
|-----|----------|
| 1️⃣ | Определяет ОС (Ubuntu / Debian / CentOS / Rocky / Alma) |
| 2️⃣ | Анализирует CPU, RAM и пропускную способность сети |
| 3️⃣ | Рассчитывает максимальное число пользователей |
| 4️⃣ | Устанавливает зависимости и собирает MTProxy из исходников |
| 5️⃣ | Генерирует уникальный секрет для подключения |
| 6️⃣ | Создаёт `systemd`-сервис с автозапуском |
| 7️⃣ | Настраивает автообновление конфига Telegram (каждую ночь в 03:00) |
| 8️⃣ | Открывает порт 443 в файрволе (UFW / firewalld) |
| 9️⃣ | Выводит готовую ссылку для подключения |

## 📊 Расчёт мощности сервера

Скрипт автоматически считает, сколько пользователей потянет ваш сервер, исходя из трёх ограничений:

```
CPU   → каждое ядро держит ~200 одновременных соединений
RAM   → ~1 МБ на одного активного пользователя
Сеть  → ~50 КБ/с на пользователя (400 Кбит/с)

Итог  → минимум из трёх значений = реальный лимит
```

**Примеры:**

| Конфигурация | CPU лимит | RAM лимит | Сеть лимит | Итог |
|---|---|---|---|---|
| 1 ядро / 512 МБ / 100 Мбит | 200 | 410 | 250 | **~200** |
| 2 ядра / 2 ГБ / 1 Гбит | 400 | 1638 | 2500 | **~400** |
| 4 ядра / 8 ГБ / 1 Гбит | 800 | 6553 | 2500 | **~800** |
| 8 ядер / 16 ГБ / 1 Гбит | 1600 | 13107 | 2500 | **~1600** |

## 📱 Как подключиться

**Способ 1 — По ссылке (проще всего)**

После установки скопируйте ссылку из вывода скрипта и нажмите на неё — Telegram сам откроет окно подключения.

```
https://t.me/proxy?server=ВАШ_IP&port=443&secret=ВАШ_СЕКРЕТ
```

**Способ 2 — Вручную**

```
Android:  Настройки → Данные и память → Прокси
iOS:      Настройки → Данные и память → Прокси
Desktop:  Настройки → Продвинутые → Тип соединения
```

Тип: **MTProto** · Сервер: `ВАШ_IP` · Порт: `443` · Секрет: `ВАШ_СЕКРЕТ`

## 🛠️ Управление сервером

```bash
# Статус
systemctl status mtproxy

# Логи в реальном времени
journalctl -u mtproxy -f

# Перезапуск
systemctl restart mtproxy

# Остановить
systemctl stop mtproxy

# Посмотреть секрет
cat /opt/mtproxy/secret.txt
```

## ❓ FAQ

<details>
<summary><b>Прокси может читать мои сообщения?</b></summary>

Нет. Весь трафик Telegram зашифрован на стороне клиента. Оператор прокси видит только факт подключения и IP-адрес, но не содержимое переписки.

</details>

<details>
<summary><b>Почему порт 443?</b></summary>

Порт 443 — стандартный HTTPS. Трафик на него выглядит как обычный веб-сёрфинг, что сильно затрудняет блокировку со стороны провайдеров.

</details>

<details>
<summary><b>Нужен ли сервер за рубежом?</b></summary>

Не обязательно. Российские VPS нередко работают стабильнее зарубежных, так как фильтрация активнее применяется к трансграничному трафику. Главное — чтобы сервер имел доступ к серверам Telegram.

</details>

<details>
<summary><b>Как обновить конфигурацию вручную?</b></summary>

```bash
curl -s https://core.telegram.org/getProxyConfig -o /opt/mtproxy/proxy-multi.conf
systemctl restart mtproxy
```

</details>

<details>
<summary><b>Прокси перестал работать после обновления системы?</b></summary>

```bash
systemctl restart mtproxy
journalctl -u mtproxy -n 50
```

Если ошибки в логах — пересоберите бинарник, запустив скрипт установки повторно.

</details>
